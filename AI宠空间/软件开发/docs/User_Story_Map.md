# PetCare AI 宠护平台 - 用户故事地图 (User Story Map)

## 1. 故事地图概览
- **愿景**: 极致宠护预约体验。
- **业务流程 (Backbone)**: 
  - 1. 账户建立 (Onboarding)
  - 2. 服务选择 (Selection)
  - 3. 预约提交 (Booking)
  - 4. 状态反馈 (Feedback)
  - 5. 管理调度 (Administration)
  - 6. 统计决策 (Insights)

## 2. 详细地图

| 用户活动 (Activities) | 任务 (Tasks) | 故事描述 (User Stories) | 版本 (Release) |
| :--- | :--- | :--- | :--- |
| **账户建立** | 身份校验 | 作为 Alice，我能通过手机验证码快速登录，以便无需设置复杂密码。 | MVP (v1.0) |
| | 宠物档案 | 作为 Alice，我能录入宠物昵称，以便为多只宠物预约服务。 | MVP (v1.0) |
| **服务选择** | 选择核心服务 | 作为 Alice，我能从基础服务列表中选择，以便明确服务范围。 | MVP (v1.0) |
| | 勾选增值项 | 作为 Alice，我能勾选剪趾甲、洗耳等增值项，以便享受精细化服务。 | MVP (v1.0) |
| | 实时价格确认 | 作为 Alice，我能在下单前看到明细总价，以便确认预算。 | MVP (v1.0) |
| **预约提交** | 极速下单 | 作为 Alice，我能点击一键预约，以便在 3 秒内完成流程。 | MVP (v1.0) |
| **状态反馈** | 实时追踪 | 作为 Alice，我能在“最近订单”中看到待派单、处理中状态，以便放心。 | MVP (v1.0) |
| | 历史订单 | 作为 Alice，我能查看过去的消费记录，以便快速复购。 | v1.5 |
| **管理调度** | 仪表盘监控 | 作为 Bob，我能看到实时订单动态与核心 KPI，以便掌握全局。 | MVP (v1.0) |
| | 订单分派 | 作为 Bob，我能手动将订单指派给特定员工，以便灵活调度。 | MVP (v1.0) |
| | 状态手动流转 | 作为 Bob，我能点击按钮将订单推进至下一阶段，以便同步进度。 | MVP (v1.0) |
| | 智能派单 | 作为 Bob，我希望能系统自动派单，以便减少人工干预。 | v1.5 |
| **统计决策** | 预算偏差分析 | 作为 Bob，我能对比预估价格与实际支出，以便控制成本。 | v1.5 |
| | 用户生命周期分析 | 作为 Bob，我能识别高净值与流失用户，以便精准运营。 | v2.0 |
| | 智能建议 Agent | 作为 Bob，我希望 AI 能自动提示人员扩充或促销，以便优化经营。 | v2.0 |

## 3. 版本映射 (Mapping)
- **MVP (v1.0)**: 满足 Alice 预约、Bob 监控与手动调度的最小闭环。
- **v1.5**: 引入自动化与分析能力，提升运营效率。
- **v2.0**: 引入 AI 智能体协作，实现数据驱动的智能经营。
