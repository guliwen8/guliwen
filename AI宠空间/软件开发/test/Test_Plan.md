wty# PetCare AI 宠护平台 - 全面测试计划 (Test Plan)

## 1. 项目概述
### 1.1 项目背景
PetCare AI 宠护平台是一个面向城市养宠人群的高频低价、极简交互的标准化宠护服务平台。平台覆盖 Web、Flutter 移动端、Taro 小程序及后端 API。

### 1.2 测试目标
- 验证核心业务流程（极速预约、用户认证）的正确性。
  - 用户认证需支持“账号+密码”登录/注册，并移除短信验证码强制校验。
  - 登录入口需统一在导航栏右上方。
- 确保多端（Web, Flutter, Taro）的一致性与兼容性。
- 评估系统在高并发下的性能表现。
- 确保系统符合安全规范（OWASP Top 10）。

### 1.3 测试范围
- **后端**: API 接口、业务逻辑、数据库交互、缓存处理。
- **前端 Web**: 响应式布局、交互组件、状态管理。
- **Flutter 移动端**: iOS/Android 原生体验、API 集成、性能。
- **Taro 小程序**: 微信/支付宝环境兼容性、平台特定 API。

## 2. 测试策略
### 2.1 测试分层
- **单元测试 (Unit Testing)**: 验证最小功能单元（函数、组件）。
- **集成测试 (Integration Testing)**: 验证模块间的协作，特别是核心业务流。
- **系统测试 (System Testing)**: 端到端 (E2E) 验证完整功能。
- **非功能测试**: 性能测试、安全测试、兼容性测试。

### 2.2 测试工具
- **单元测试**: Jest (Node.js/React/Taro), flutter_test (Flutter)。
- **接口测试**: Postman, Supertest.
- **性能测试**: k6 (推荐) 或 JMeter。
- **安全测试**: OWASP ZAP, Burp Suite。
- **自动化 E2E**: Playwright (Web), Patrol (Flutter)。

## 4. 核心业务流程集成测试

### 4.1 用户认证流程 (User Auth)
1.  **场景**: 首次登录。
2.  **步骤**:
    - 调用 `POST /auth/otp` 获取验证码。
    - 验证码存入 Redis。
    - 调用 `POST /auth/login` 提交手机号及验证码。
3.  **验证点**:
    - 后端生成有效的 JWT。
    - 前端持久化存储 Token。
    - 后续请求 Header 自动带上 `Authorization: Bearer <token>`。

### 4.2 极速预约流程 (Express Appointment)
1.  **场景**: 用户在首页 3 步内完成预约。
2.  **步骤**:
    - 调用 `GET /services` 获取最新价格表。
    - 用户填写宠物名并勾选增值项。
    - 调用 `POST /bookings` 提交订单。
3.  **验证点**:
    - 数据库生成状态为 `pending` 的订单。
    - 后端实时计算的价格与前端显示一致。
    - 管理后台实时收到新订单通知。

## 5. 性能测试方案 (k6)

### 5.1 测试场景
- **负载测试 (Load Test)**: 模拟 100 VUs (虚拟用户) 持续 10 分钟，验证预约接口响应。
- **压力测试 (Stress Test)**: 逐步增加至 1000 VUs，寻找系统崩溃点。
- **峰值测试 (Spike Test)**: 模拟突发流量（如营销活动启动）。

### 5.2 性能指标 (SLA)
- **响应时间 (P95)**: < 200ms。
- **错误率**: < 0.1%。
- **吞吐量 (TPS)**: > 500 次/秒。

## 6. 安全测试方案 (OWASP Top 10)

| 风险类别 | 测试手段 | 验证点 |
| :--- | :--- | :--- |
| **A01: 权限控制** | 越权访问 (IDOR) | A 用户无法通过修改 `booking_id` 查看 B 用户的订单。 |
| **A03: 注入攻击** | SQL 注入测试 | 在查询参数中尝试输入 `' OR 1=1 --`。 |
| **A07: 身份认证失败** | 暴力破解 OTP | 验证同一手机号短时间内获取 OTP 的频率限制。 |
| **A08: 软件和数据完整性** | JWT 伪造 | 尝试使用弱密钥签名 JWT 或修改 Payload。 |
